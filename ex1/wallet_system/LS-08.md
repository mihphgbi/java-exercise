# LS-08: Wallet System Test Cases

## Tester Objective

As a Tester, verify the main Wallet System edge cases with a simple, maintainable test set.

## Scope

The test cases cover:

- Deposit calculation
- Invalid deposit input
- Large balance values
- Average spending calculation
- Divide-by-zero prevention
- Output formatting
- Recovery after invalid input

## Test Cases

| Test Case ID | Test Scenario | Test Data | Expected Result | Priority |
|---|---|---|---|---|
| TC-WS-001 | Deposit valid amount | Balance: `0`, Deposit: `500000` | Balance becomes `500000 VND`. | High |
| TC-WS-002 | Deposit zero amount | Balance: `500000`, Deposit: `0` | Deposit is rejected. | High |
| TC-WS-003 | Deposit negative amount | Balance: `500000`, Deposit: `-1` | Deposit is rejected. | Critical |
| TC-WS-004 | Deposit with invalid text | Balance: `0`, Deposit: `abc` | Input is rejected with a clear error. | Critical |
| TC-WS-005 | Deposit with blank input | Balance: `0`, Deposit: blank | Input is rejected with a clear error. | High |
| TC-WS-006 | Deposit with spaces around number | Balance: `0`, Deposit: ` 500000 ` | Spaces are trimmed and balance becomes `500000 VND`. | Medium |
| TC-WS-007 | Deposit with decimal or comma format | Deposit: `1000.50` or `1,000` | Input is rejected because only whole numbers are supported. | High |
| TC-WS-008 | Deposit when current balance is negative | Balance: `-1`, Deposit: `100000` | Current balance is rejected. | Critical |
| TC-WS-009 | Deposit over old integer limit | Balance: `2147000000`, Deposit: `1000000` | Balance becomes `2148000000 VND`; no integer overflow occurs. | Critical |
| TC-WS-010 | Average spending with valid values | Total spent: `1000000`, Transactions: `4` | Average spending is `250000 VND`. | High |
| TC-WS-011 | Average spending with zero total spent | Total spent: `0`, Transactions: `5` | Average spending is `0 VND`. | Medium |
| TC-WS-012 | Average spending with zero transactions | Total spent: `0`, Transactions: `0` | Calculation is rejected; app must not divide by zero. | Critical |
| TC-WS-013 | Average spending with invalid values | Total spent: `abc`, Transactions: `5`; or Total spent: `500000`, Transactions: `abc` | Input is rejected with a clear error. | Critical |
| TC-WS-014 | Average spending with remainder | Total spent: `1000000`, Transactions: `3` | Average spending is `333333.33 VND`. | Medium |
| TC-WS-015 | Output and recovery check | Invalid input followed by valid input | App reports the invalid input, then still handles the valid input correctly. | High |

## Risk Mapping

| Risk | Covered By |
|---|---|
| Invalid money input causes crash or bad calculation. | TC-WS-002, TC-WS-003, TC-WS-004, TC-WS-005, TC-WS-007, TC-WS-013 |
| Balance overflow creates an incorrect balance. | TC-WS-009 |
| Average spending divides by zero. | TC-WS-012 |
| Negative values create invalid wallet state. | TC-WS-003, TC-WS-008 |
| Output is unclear or app cannot recover after invalid input. | TC-WS-015 |

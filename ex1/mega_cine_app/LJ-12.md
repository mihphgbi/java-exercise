# LJ-12: TicketPricingService Test Cases

## Tester Objective

As a Tester, verify that `TicketPricingService.calculatePrice(age, dayOfWeek, isMember)` returns the correct ticket price for valid customer groups, membership status, day rules, and invalid input.

## Pricing Rules Observed From Current Code

- Base ticket price is `100000`.
- Invalid `age <= 0` returns `-1`.
- Invalid `dayOfWeek == null` or empty string `""` returns `-1`.
- `Tuesday` has highest priority and returns `50000` for any valid age and membership status.
- Child age is `1..12`: normal price is `70000`; member price is `60000`.
- Senior age is `60+`: price is `60000`; membership does not change senior price.
- Adult age is `13..59`.
- Adult weekend days `Saturday` and `Sunday`: normal price is `120000`; member price is `108000`.
- Adult non-weekend, non-Tuesday days: normal price is `100000`; member price is `90000`.
- Day comparison is case-insensitive.
- Day strings are not trimmed; whitespace-only or padded days are treated as ordinary non-weekend, non-Tuesday text unless empty.

## Edge Case Summary

| Edge Area | Cases |
|---|---|
| Age boundaries | `0`, `1`, `12`, `13`, `59`, `60`, `120`, `Integer.MAX_VALUE`, `Integer.MIN_VALUE` |
| Day validation | `null`, `""`, `" "`, padded valid days, unknown non-empty day |
| Day case handling | lowercase and mixed-case `Tuesday`, `Saturday`, `Sunday` |
| Rule priority | Tuesday before child, senior, member, and weekend rules |
| Membership impact | child discount, adult weekday discount, adult weekend discount, no senior discount |
| Weekend behavior | adult surcharge only; child and senior prices ignore weekend surcharge |
| Return type | expected prices are returned as `double` values |
